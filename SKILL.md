# 风格描述提示词

## 概述

将输入图片转为高通用、可复用的“风格描述提示词”。
严格按固定结构输出，保证后续可直接复制到任意文生图/图生图工作流中。

## 触发条件

在以下情况触发本技能：
- 用户给出图片并要求“分析画风/风格拆解/提示词生成”。
- 用户要求使用固定模板返回（全局、角色、环境、风格介质、关系、排除项）。
- 用户要求可量化、可复用、通用性强的风格描述。

## 工作流程

1. 先识别画面类型：角色主导、环境主导、均衡、或无明确角色。
2. 按 `references/api_reference.md` 的跨介质量表规则给关键维度打分（0-100）。
3. 用短语填充语义槽位（风格名、空气感、主色调、天气/时间、光源、介质倾向、排除项）。
4. 进行一致性校验：角色与背景分离逻辑、光源逻辑、材质响应、统一度是否冲突。
5. 仅输出最终模板结果；除非用户明确要求，否则不附加分析过程。

## 输出规则

- 始终使用中文输出。
- 始终保留以下六个分区标题，顺序不得改变。
- 分值统一使用 `x/100`。
- 无法确定时给“中性值+低风险描述”：默认 `50/100`，并避免极端判断。
- 画面无角色时，将角色段写为“主体（非角色）”，并保持结构不变。
- 排除项给 3 个，优先写最常见失败项（如过曝、脏灰、材质冲突、光源错位、塑料感等）。

## 泛化约束

- 不绑定具体引擎参数（不写模型私有字段），确保跨平台复用。
- 描述优先使用“可观察视觉证据”，避免依赖题材语义。
- 对二次元、插画、游戏原画、写实摄影、3D渲染、电影感合成图均可适配。
- 写实与3D场景优先保证光源、材质、镜头语言一致，不强制附加手绘特征。
- 若用户额外要求“更像参考图”，优先保持结构与关系项，不优先堆砌形容词。
